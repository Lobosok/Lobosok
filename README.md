Введение
Проект «HR-помощник» представляет собой инновационную цифровую платформу, разработанную для полной автоматизации и оптимизации процессов управления человеческими ресурсами в современных компаниях. В эпоху цифровизации и ускоряющихся изменений на рынке труда, где компании сталкиваются с дефицитом квалифицированных кадров, высокой текучестью персонала и необходимостью быстрого масштабирования, традиционные методы HR-работы становятся неэффективными. Эта система интегрирует искусственный интеллект, чат-боты и аналитические инструменты для поддержки всех этапов работы с сотрудниками — от рекрутинга до развития и удержания талантов. 

Цель
Разработка виртуального HR-помощника для автоматизации процессов подбора персонала.

Актуальность
Актуальность проекта HR-помощник в 2025 году подтверждается масштабными изменениями и новыми требованиями в сфере управления персоналом, вызванными цифровизацией, развитием искусственного интеллекта и изменениями на рынке труда. Современные компании сталкиваются с необходимостью быстро и эффективно адаптироваться к этим переменам, чтобы сохранять конкурентоспособность и удерживать таланты.

Технологический этап
Архитектура "HR-помощника" строится на основе многоуровневой модели, обеспечивающей чёткое разделение ответственности между компонентами системы. В основе проектирования лежит принцип модульности, позволяющий выделить ключевые функциональные блоки и определить взаимодействие между ними. Система организована в виде трёх основных уровней: уровень представления, уровень бизнес-логики и уровень данных, каждый из которых выполняет строго определённые функции и взаимодействует с соседними уровнями через чётко определённые интерфейсы.

Уровень представления
Этот уровень отвечает за взаимодействие с сотрудником через интерфейсы, такие как чат в Telegram, отображая информацию о списках вакансий. Когда кто то откликнулся на вакансию бот добавляет этого человека в базу данных. Дальше администратор бота видит этого кандидата и звонит ему и приглашает на собеседование.

База данных 
Используется две основные таблицы:
- **vacancies**: Хранит список открытых позиций (`id`, `title`, `description`).
- **candidates**: Хранит отклики (`id`, `name`, `phone`, `resume`).
Логика бота
**ConversationHandler**: Реализует пошаговую анкету кандидата (Имя -> Телефон -> Резюме).
- **InlineButtons**: Интерактивное меню для выбора вакансий и подачи заявки.
- **Admin Protection**: Декоратор `@admin_only` ограничивает доступ к командам управления (`/add_vacancy`, `/list_candidates`) только для указанного `ADMIN_ID`
Сценарий использования
 **HR-менеджер:**
  - Добавляет вакансии через команду `/add_vacancy`.
  - Просматривает список всех откликов через `/list_candidates`.
- **Кандидат:**
  - Заходит в бота, видит список вакансий.
  - Нажимает "Откликнуться".
  - Бот последовательно запрашивает данные.
  - Данные сохраняются в БД.
